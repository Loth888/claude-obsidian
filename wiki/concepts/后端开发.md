# 后端开发

## 定义

后端开发是构建**用户看不到但至关重要的服务端逻辑**的技术领域。它负责接收请求、处理数据、调用外部服务（如数据库、API），并将结果返回给前端。

刘润的比喻：
> "后端就像饭店的后厨，用户看不到，但也很重要。负责接收请求、处理数据、调用数据库、再返回结果给前端。"

## 核心要点

### 1. 后端的核心职责

| 职责 | 说明 |
|------|------|
| **接收请求** | 接收前端发送的数据和指令 |
| **处理数据** | 验证、转换、计算数据 |
| **调用服务** | 访问数据库、调用外部API |
| **安全控制** | 身份验证、权限管理、敏感信息保护 |
| **返回结果** | 将处理结果返回给前端 |

### 2. 为什么AI编程需要后端？

**场景**：工具需要调用DeepSeek API

**问题**：
- ❌ 前端直接调用 → API Key暴露在浏览器中
- ❌ 任何人都能窃取Key → 账户被盗刷

**解决**：
- ✅ 后端代理调用 → Key安全存储在服务器
- ✅ 前端只与后端通信 → 不接触敏感信息

### 3. 后端代理的工作流程

```
用户操作前端界面
    ↓
前端发送请求到后端（不含API Key）
    ↓
后端接收请求，添加API Key
    ↓
后端调用DeepSeek API
    ↓
后端接收AI返回结果
    ↓
后端转发结果给前端
    ↓
前端展示给用户
```

### 4. 常用后端技术

| 技术 | 特点 | 适用场景 |
|------|------|----------|
| **Python** | 简单易学，AI生态丰富 | AI编程首选 |
| **Node.js** | JavaScript全栈 | 前后端统一语言 |
| **Java** | 企业级，稳定 | 大型系统 |
| **Go** | 高性能，并发强 | 高并发服务 |

### 5. AI编程中的后端

在刘润的教程中，后端是一个**Python代理脚本**：
- 接收前端发送的客户数据
- 调用DeepSeek API进行分析
- 返回AI生成的评分和建议

**示例代码结构**：
```python
# 后端代理服务
from flask import Flask, request
import requests

app = Flask(__name__)
API_KEY = "sk-xxxxxx"  # 安全存储在服务器

@app.route('/analyze', methods=['POST'])
def analyze():
    data = request.json
    # 调用DeepSeek API
    response = requests.post(
        'https://api.deepseek.com/v1/chat',
        headers={'Authorization': f'Bearer {API_KEY}'},
        json={'messages': [...]}
    )
    return response.json()
```

## 实践应用

### 场景：保护API Key的后端代理

**前端代码**（安全）：
```javascript
// 只发送数据，不含Key
fetch('/analyze', {
  method: 'POST',
  body: JSON.stringify({customer: '张三'})
})
```

**后端代码**（安全存储Key）：
```python
# Key只在服务器端，用户看不到
API_KEY = os.environ.get('DEEPSEEK_API_KEY')
```

## 前端与后端的协作

| 阶段 | 前端 | 后端 |
|------|------|------|
| 用户输入 | 收集表单数据 | - |
| 发送请求 | 发送数据到后端 | 接收请求 |
| 处理数据 | - | 验证、处理、调用API |
| 展示结果 | 接收并展示 | 返回处理结果 |
