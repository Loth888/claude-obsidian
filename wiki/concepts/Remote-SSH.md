# Remote - SSH

## 定义

Remote - SSH 是 VSCode 的一个插件，让你可以**远程连接到服务器**并在本地编辑器中操作。就像坐在服务器面前一样编写代码、运行程序，无需在服务器上安装图形界面。

## 核心要点

### 1. 为什么需要 Remote - SSH？

**场景**：购买了云服务器，需要部署程序

**传统方式的问题**：
- ❌ 命令行操作，不直观
- ❌ 无法使用熟悉的编辑器
- ❌ 文件传输麻烦

**Remote - SSH 的优势**：
- ✅ 在本地 VSCode 中编辑服务器上的文件
- ✅ 使用所有熟悉的插件和功能
- ✅ 图形化界面操作服务器
- ✅ 文件自动同步

### 2. 安装与配置

**安装插件**：
1. 打开 VSCode
2. 点击左侧插件市场图标（四个小方块）
3. 搜索 "Remote - SSH"
4. 点击安装

**添加服务器配置**：
1. 安装后左侧会出现远程图标
2. 点击 "+" 添加新连接
3. 输入连接命令：
   ```
   ssh root@你的公网IP
   ```
   例如：`ssh root@118.25.194.112`

### 3. 连接服务器

**连接步骤**：
1. 点击配置好的服务器
2. 输入密码（购买服务器时获得）
3. 选择操作系统类型（Linux）
4. 等待连接成功

**连接成功后**：
- 左下角显示 SSH:IP地址
- 可以打开服务器上的文件夹
- 像在本地一样编辑文件
- 可以在服务器终端运行命令

### 4. 常用操作

| 操作 | 方法 |
|------|------|
| **打开文件夹** | File → Open Folder（服务器上的路径） |
| **新建文件** | 右键 → New File |
| **打开终端** | Terminal → New Terminal |
| **拖拽上传** | 本地文件拖到服务器文件夹 |

## 实践应用

### AI编程中的使用场景

**部署流程**：
1. **本地开发** - 在本地 VSCode 编写和测试
2. **购买服务器** - 获取公网IP和密码
3. **连接服务器** - 使用 Remote - SSH
4. **上传代码** - 把项目文件夹拖到服务器
5. **安装依赖** - 在服务器终端运行安装命令
6. **启动服务** - 运行后端程序
7. **公网访问** - 通过IP访问你的工具

### 示例：部署客户评分工具

```bash
# 在 Remote - SSH 连接的终端中执行

# 1. 进入项目目录
cd /root/projects

# 2. 创建项目文件夹
mkdir customer-scoring
cd customer-scoring

# 3. 安装Python依赖（AI会帮你生成requirements.txt）
pip install flask requests

# 4. 启动服务
python app.py

# 5. 现在可以通过 http://你的IP:5000 访问了
```

## 连接信息

**SSH 连接格式**：
```
ssh 用户名@公网IP
```

**示例**：
```
ssh root@118.25.194.112
```

**需要的信息**：
- **用户名**：通常是 `root`（管理员）
- **公网IP**：服务器的互联网地址
- **密码**：购买服务器时设置的密码

## 常见问题

### 连接失败

**可能原因**：
- IP地址错误
- 密码错误
- 服务器防火墙阻止
- SSH服务未启动

**解决**：
- 检查IP和密码
- 在云服务商控制台检查安全组规则
- 确认22端口开放

### 文件权限错误

**现象**：无法保存文件

**解决**：
```bash
# 修改文件权限
chmod 755 文件夹名
```
