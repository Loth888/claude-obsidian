# 大模型

## 定义

大模型（Large Language Model，LLM）是**基于海量数据训练的人工智能模型**，具有强大的语言理解和生成能力。它能理解自然语言、回答问题、生成文本、编写代码，是当前AI应用的核心技术。

刘润的比喻：
> "API里流出来的，是一token一token的'智能'。从这个角度说，各家大模型，做的其实是'发电厂'的生意。"

## 核心要点

### 1. 什么是"大"？

| 维度 | 说明 | 规模 |
|------|------|------|
| **参数量** | 模型内部可调节的参数 | 数十亿到数万亿 |
| **训练数据** | 学习用的文本数据 | 数万亿token |
| **计算资源** | 训练所需的算力 | 数千GPU数月 |

### 2. 大模型的能力

**通用能力**：
- ✅ 理解自然语言
- ✅ 生成流畅文本
- ✅ 回答问题
- ✅ 翻译语言
- ✅ 总结内容

**专业能力**：
- ✅ 编写代码
- ✅ 分析数据
- ✅ 创作内容
- ✅ 逻辑推理
- ✅ 数学计算

### 3. 主流大模型

| 模型 | 厂商 | 特点 | 接入方式 |
|------|------|------|----------|
| **DeepSeek** | 深度求索 | 国产，性价比高，推理强 | API |
| **GPT-4** | OpenAI | 综合能力最强 | API / ChatGPT |
| **Claude** | Anthropic | 长文本，安全性高 | API / Claude Code |
| **Kimi** | Moonshot | 国产，长上下文 | Kimi Code |
| **Gemini** | Google | 多模态，谷歌生态 | API / Gemini Code |

### 4. 如何使用大模型？

**方式一：直接使用**（如ChatGPT）
- 打开网页或App
- 输入问题
- 获得回答

**方式二：API调用**（AI编程场景）
- 申请API Key
- 通过代码调用
- 集成到自己的工具中

**API调用示例**：
```python
import requests

response = requests.post(
    'https://api.deepseek.com/v1/chat',
    headers={'Authorization': 'Bearer sk-xxxxxx'},
    json={
        'messages': [
            {'role': 'user', 'content': '分析这个客户数据...'}
        ]
    }
)

result = response.json()
```

### 5. Token是什么？

**定义**：大模型处理文本的基本单位

**类比**：
- 中文：1个汉字 ≈ 1-2个token
- 英文：1个单词 ≈ 1-2个token

**计费方式**：
- 按输入+输出的token总数计费
- 通常：1元 = 数万token

刘润的比喻：
> "一token一token的'智能'"

## 实践应用

### AI编程中的大模型

**场景**：客户评分工具需要"智能"分析

**实现步骤**：
1. **用户输入** - 客户信息
2. **前端** - 收集数据，发送到后端
3. **后端** - 调用大模型API
4. **大模型** - 分析并生成评分和建议
5. **后端** - 返回结果给前端
6. **前端** - 展示给用户

**关键**：大模型让工具"懂"业务逻辑

### 选择合适的模型

| 需求 | 推荐模型 | 原因 |
|------|----------|------|
| 代码生成 | Claude / GPT-4 | 代码能力强 |
| 中文场景 | DeepSeek / Kimi | 中文优化好 |
| 长文档处理 | Kimi / Claude | 上下文长 |
| 成本敏感 | DeepSeek | 性价比高 |

## 大模型的局限性

**不是万能的**：
- ❌ 不知道你的具体业务
- ❌ 无法访问实时信息（除非联网）
- ❌ 可能产生幻觉（编造信息）
- ❌ 需要明确的指令

**正确的使用方式**：
> "别把AI当专家。把它当一个博士水平的实习生。"
> 
> "你要首先懂你的业务。然后，手把手地教给它。"
