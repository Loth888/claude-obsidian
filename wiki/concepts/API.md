# API

## 定义

API（Application Programming Interface，应用程序接口）是**不同软件系统之间通信的桥梁**。它定义了程序之间如何互相请求服务和交换数据的标准方式。

刘润的比喻：
> "API就像电线。电线里流出来的是一度一度的电，API里流出来的是一token一token的'智能'。"

## 核心要点

### 1. 为什么需要API？

**场景**：你的工具需要AI能力来分析客户数据
- ❌ 自己训练AI模型：成本高、技术难
- ✅ 调用大模型API：低成本、即插即用

### 2. API的组成

| 组件 | 说明 | 示例 |
|------|------|------|
| **Endpoint** | 接口地址 | `https://api.deepseek.com/v1/chat` |
| **Method** | 请求方式 | GET、POST、PUT、DELETE |
| **Headers** | 请求头信息 | 认证信息、内容类型 |
| **Body** | 请求体 | 要发送的数据 |
| **Response** | 响应结果 | 返回的数据 |

### 3. API Key（密钥）

**作用**：身份认证和权限控制
**重要性**：⚠️ **极度重要**

> "API Key是你调用能力的钥匙。如果泄露，就相当于把信用卡丢在大街上。"

**安全建议**：
- 不要硬编码在前端代码中
- 使用环境变量或后端代理
- 设置额度上限
- 定期更换

### 4. 前端直接调用 vs 后端代理

| 方式 | 优点 | 缺点 | 安全性 |
|------|------|------|--------|
| 前端直接调用 | 简单快速 | API Key暴露 | ❌ 不安全 |
| 后端代理 | Key安全隐藏 | 需要搭建后端 | ✅ 安全 |

**推荐**：生产环境必须使用后端代理

## 实践应用

### 接入DeepSeek API的流程

1. **注册账号**：https://platform.deepseek.com/
2. **充值预存**：十几二十块足够测试
3. **创建API Key**：在控制台生成
4. **配置后端代理**：保护Key不被泄露
5. **调用接口**：发送请求获取AI能力

### 典型API调用场景

```
用户上传客户数据
    ↓
前端发送到后端
    ↓
后端调用DeepSeek API
    ↓
AI分析并返回结果
    ↓
后端转发给前端展示
```
