# 云服务器部署

## 定义

云服务器部署是指将**本地开发的程序上传到云服务器并运行**，使其可以通过互联网被任何人访问的过程。这是AI编程的最后一步，让工具从"本地可用"变成"全球可访问"。

## 核心要点

### 1. 部署的目标

**本地开发的问题**：
- ❌ 只有你的电脑能访问
- ❌ 关机后程序停止
- ❌ 同事和客户打不开

**部署后的效果**：
- ✅ 24小时在线运行
- ✅ 有固定公网地址
- ✅ 任何人都能访问
- ✅ 团队协作可用

### 2. 部署流程（AI编程场景）

```
本地开发完成
    ↓
购买云服务器
    ↓
远程连接服务器（SSH）
    ↓
在服务器上安装环境
    ↓
上传项目代码
    ↓
配置并启动服务
    ↓
通过公网IP访问
    ↓
完成部署 ✅
```

### 3. 详细步骤

#### Step 1: 购买云服务器
- 选择：腾讯云/阿里云/华为云
- 配置：最低配（1核2G）足够
- 系统：Debian/Ubuntu（Linux）
- 获取：公网IP、登录密码

#### Step 2: 远程连接
- 使用 VSCode + Remote - SSH
- 或命令行：`ssh root@你的IP`

#### Step 3: 安装环境
```bash
# 更新系统
apt update && apt upgrade -y

# 安装Python
apt install python3 python3-pip -y

# 安装Node.js（如需要）
apt install nodejs npm -y
```

#### Step 4: 上传代码
- 方法1：VSCode Remote 直接拖拽
- 方法2：SCP命令：`scp -r 本地文件夹 root@IP:/服务器路径`
- 方法3：Git克隆：`git clone 你的仓库地址`

#### Step 5: 安装依赖
```bash
cd /你的项目路径
pip install -r requirements.txt  # Python
npm install                     # Node.js
```

#### Step 6: 启动服务
```bash
# Python Flask
python3 app.py

# Node.js
node server.js

# 或使用进程管理器（推荐）
nohup python3 app.py &  # 后台运行
```

#### Step 7: 访问测试
- 浏览器访问：`http://你的IP:端口号`
- 例如：`http://118.25.194.112:5000`

### 4. 部署方式对比

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **直接运行** | 简单 | 关闭终端就停止 | 临时测试 |
| **nohup** | 后台运行 | 重启后需手动启动 | 小项目 |
| **systemd** | 开机自启 | 配置稍复杂 | 生产环境 |
| **Docker** | 环境隔离 | 学习成本 | 复杂项目 |
| **Nginx** | 高性能 | 配置复杂 | 高并发 |

## 常见问题

### 端口访问不了

**原因**：防火墙/安全组未开放端口

**解决**：
```bash
# 开放端口
ufw allow 5000

# 或在云服务商控制台配置安全组
```

### 程序启动后断开

**原因**：没有后台运行

**解决**：
```bash
# 使用nohup后台运行
nohup python3 app.py &

# 或使用screen
screen -S myapp
python3 app.py
# Ctrl+A+D 退出，程序继续运行
```

### 依赖安装失败

**原因**：网络问题或权限问题

**解决**：
```bash
# 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或使用sudo
sudo pip install xxx
```

## 进阶：域名和HTTPS

### 绑定域名
1. 购买域名（阿里云/腾讯云）
2. 添加DNS解析 → 指向服务器IP
3. 配置Nginx反向代理

### 配置HTTPS
1. 申请SSL证书（Let's Encrypt免费）
2. 配置Nginx使用证书
3. 强制HTTPS访问
